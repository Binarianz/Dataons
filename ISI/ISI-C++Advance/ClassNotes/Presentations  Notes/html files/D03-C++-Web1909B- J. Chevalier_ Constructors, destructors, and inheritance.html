Constructors, destructors, and inheritance

In the previous example, we did not use a constructor or a destructor. It is time to address this important aspect of OOP. Firstly, as was already mentioned (Section 2.3.2), constructors and destructors are not inherited, but rather are called by constructors and destructors of derived classes. So, a derived class inherits all members, attributes, and methods from its base class except for constructors and destructors. Instead, the constructors and destructors of derived classes call those of the base class. 

To illustrate this, we will define two classes (a base class and a child class), that contain only a default constructor and destructor. Since there are no attributes that need to be initialized, the constructor and destructor have nothing to do. We will therefore just display a message in order to see the order in which the constructors and destructors are called. Take a good look at this example, and then examine the displayed results. The ordering of the messages is not arbitrary. Note that there is nothing explicit that indicates, a priori, that the constructor of the child class (Father) calls the constructor of the base class (Grandfather). Also note that the name given to our instance (variable of type class) is not important, and does not appear in the display (it is just like the name of any other variable of any type).


```

#include <iostream>

using namespace std;


class Grandfather {

public:

Grandfather();

~Grandfather();

};


Grandfather::Grandfather() {

cout << "The constructor of Grandfather has been called" << endl;

}


Grandfather::~Grandfather() {

cout << "The destructor of Grandfather has been called" << endl;

}


class Father : public Grandfather {

public:

Father();

~Father();

};


Father::Father() {

cout << "The constructor of Father has been called" << endl;

}


Father::~Father() {

cout << "The destructor of Father has been called" << endl;

}



void createFather();

int main() {

createFather();

system("pause");

return 0;

}


void createFather() {

Father f; // declaration of an instance of Father

cout << "----------------------------" << endl;

}

```

The results:

The constructor of Grandfather has been called

The constructor of Father has been called

--------------------------------------

The destructor of Father has been called

The destructor of Grandfather has been called


Notice that the constructor of the base class is called before that of the derived class, whereas the opposite is true for the destructors. The logic is as follows: a foundation is built before a floor (which is supported by the former), and the floor is removed before the foundation (so that the floor does not collapse).
