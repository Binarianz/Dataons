Simple inheritance

So far, OOP has simply been presented as a different structural style of programming from procedural programming. It has perhaps seemed a little more natural or intuitive (once you understood how to use it), but that's probably about it. With inheritance, we will see an objective advantage of OOP: reusability of code. This is not to say that procedural programming does not allow for the reuse of code; it certainly does. But we will see with inheritance that reuse of code will be much easier and will save development time and lines of code, which amounts to a lot of energy. 

The principle is simple: a class (called a child class or a derived class) can inherit the attributes and methods of another class (called the base class or the parent class), and also add attributes and methods for itself.

For example, imagine a Mammal class, with the attributes of mammals (like the various body parts shared by all mammals, like lungs), as well as methods representing vital functions shared by all mammals (like breathing, etc.). We can then imagine a Dog class with its attributes (lungs, four legs, etc.), and its methods (breathing, barking, etc.), as well as as many other classes derived from Mammal as desired.

The advantage is that, rather than re-declaring the lungs attribute in the Dog, Dolphin, Horse, Cat, etc. classes, and re-defining the breathing method across all these classes as well, we can merely define them in the mammalian base class and then indicate that the other classes inherit from this mammalian class (and thus inherit attributes like the lungs and methods like breathing). In this way, methods and attributes that are common to all the derived classes are only defined once, in the base class, and not copied for every child class.


Mechanism of inheritance

Consider a Dog class, containing an integer data member called age, as well as three methods (to get the age, to set the age, and to display the task that the dog can perform):

```

class Dog {

private:

int age;

public:

void setAge(int age);

int getAge();

void run();

void display();

};


void Dog::setAge(int age) {

this->age = age;

}


int Dog::getAge() {

return this->age;

}


void Dog::run() {

cout << "The dog runs in the park!" << endl;

}


void Dog::display() {

cout << "Age: " << this->age << endl;

this->run();

}

```

Now let's create a Bird class containing the same attribute and the same methods, in addition to another integer attribute, and a method that initializes the two attributes (the one inherited from the base class, and the one that belongs to the derived class), and a method that displays these attributes.


```

class Bird {

int age;

int wingspan;

public:

void setAge(int age);

int getAge();

void setWingspan(int wingspan);

int getWingspan();

void fly();

void display();

};


void Bird::setAge(int age) {

this->age = age;

}


int Bird::getAge() {

return this->age;

}


void Bird::setWingspan(int wingspan) {

this->wingspan = wingspan;

}


int Bird::getWingspan() {

return this->wingspan;

}


void Bird::fly() {

cout << "The bird flies in the sky!" << endl;

}


void Bird::display() {

cout << "Age : " << this->age << endl;

cout << "Wingspan: " << this->wingspan << endl;

this->fly();

}

```


We can see how there is inefficient code duplication, specifically for the age data member and the getAge() and setAge() methods. To avoid this undesired duplication, we can simply define a class that is common to both the Dog and the Bird classes, which contains the age data member, and its getter and setter:

```

class Animal {

int age;

public:

void setAge(int age);

int getAge();

void display();

};


void Animal::setAge(int age) {

this->age = age;

}


int Animal::getAge() {

return this->age;

}


void Animal::display() {

cout << "Age: " << this->age << endl;

}


class Dog : public Animal {

public:

void run();

void display();

};


void Dog::run() {

cout << "The dog runs in the park!" << endl;

}


void Dog::display() {

cout << "Age: " << this->age << endl;

this->run();

}


class Bird : public Animal {

int wingspan;

public:

void setWingspan(int wingspan);

int getWingspan();

void fly();

void display();

};


void Bird::setWingspan(int wingspan) {

this->wingspan = wingspan;

}


int Bird::getWingspan() {

return this->wingspan;

}


void Bird::fly() {

cout << "The bird flies in the sky!" << endl;

}


void Bird::display() {

cout << "Age: " << this->age << endl;

cout << "Wingspan: " << this->wingspan << endl;

this->fly();

}


```

With this technique, we no longer need to define the code of the methods inherited from Animal, since they are already defined in the Animal class itself. We only have to define code for the new methods of the derived classes. However, if you try to execute this code, you will get an error, because the age data member is no longer accessible in the display() methods of the Dog and Bird classes.


Why do we get this error? Because age is a private attribute of the Animal class. Therefore, it is accessible only to the methods of the Animal class, not to the methods of the Dog and Bird classes. This shows us that classes do not have access to the private members of their bases classes, even though they inherit them. The child classes must go through the non-private methods of the Animals class to access inherited private members. This is a bit like someone who inherits an amount in a trust: the money belongs to them, but they can only access it by going through the trust (in this way, they can ensure only a predetermined amount is removed over each fixed period, and prevent all the money from being withdrawn at once).

How do we fix this error? By using a public method in the Animal class (which has access to age). The display methods of Dog and Bird will have the following form:

```

void Dog::display() {

cout << "Age: " << this->getAge() << endl;

this->run();

}


void Bird::display() {

cout << "Age: " << this->getAge() << endl;

cout << "Wingspan: " << this->wingspan << endl;

this->fly();

}

```

We can even call the Animal display() method directly like this:

```

void Dog::display() {

Animal::display();

this->run();

}


void Bird::display() {

Animal::display();

cout << "Wingspan: " << this->wingspan << endl;

this->fly();

}


```

If we put this all together, declare an instance of the derived class, and then call its methods, we could have a program that looks like this:

```

#include <iostream>

using namespace std;


class Animal {

int age;

public:

void setAge(int age);

int getAge();

void display();

};


void Animal::setAge(int age) {

this->age = age;

}


int Animal::getAge() {

return this->age;

}


void Animal::display() {

cout << "Age: " << this->age << endl;

}


class Dog : public Animal {

public:

void run();

void display();

};


void Dog::run() {

cout << "The dog runs in the park!" << endl;

}


void Dog::display() {

Animal::display();

this->run();

}


class Bird : public Animal {

int wingspan;

public:

void setWingspan(int wingspan);

int getWingspan();

void fly();

void display();

};


void Bird::setWingspan(int wingspan) {

this->wingspan = wingspan;

}


int Bird::getWingspan() {

return this->wingspan;

}


void Bird::fly() {

cout << "The bird flies in the sky!" << endl;

}


void Bird::display() {

Animal::display();

cout << "Wingspan: " << this->wingspan << endl;

this->fly();

}



```
